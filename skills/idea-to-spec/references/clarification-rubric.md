# Clarification Rubric

Use this rubric to interrogate the user's idea before turning it into a spec.

## Core Questions

Check whether the idea clearly answers:

1. Who is the user?
2. What problem are they actually trying to solve?
3. Why is the current way insufficient?
4. What input enters the system?
5. What output should the system produce?
6. What does success look like?

## Ambiguity Traps

Flag words that often hide multiple meanings:

- "platform"
- "smart"
- "automatic"
- "manage"
- "analyze"
- "optimize"
- "agent"
- "workflow"
- "quality"

When these appear, replace them with observable behavior.

## Scope Traps

Challenge ideas that silently assume all of these at once:

- ingestion
- processing
- storage
- search
- analytics
- collaboration
- permissions
- deployment

Cut to the first valuable slice instead of keeping the full imagined system intact.

## Response Pattern

When the idea is rough, respond in this order:

1. concise restatement
2. what is strong about the idea
3. what is still vague or risky
4. proposed rewrite
5. a few high-leverage questions
