# Project Bootstrap

## Startup Sequence

Use this sequence when entering a new repo or subproject:

1. Create project memory with `init_agent_project.py` if `.codex-agent/PROJECT.md` does not exist.
2. Read `PROJECT.md` and `TASTE.md` before substantial work.
3. Inspect local `AGENTS.md`, available skills, and reusable scripts before inventing a new workflow.
4. Start a task directory with `init_agent_task.py`.
5. Fill in acceptance criteria and validation commands before implementation.

## Project Memory Standard

Keep project memory short but useful:

- architecture map
- main commands
- test entrypoints
- risky areas
- coding and product standards

Do not turn project memory into a wiki. Update it only when the information would help a future Codex session act correctly.
